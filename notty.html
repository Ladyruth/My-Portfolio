Crypto Launchpad

Notty Terminal

Streamlined interface and product design for a crypto launchpad, making complex token sales and decentralized finance accessible.

Role

UI/UX Designer

Tools

Figma

Duration

3 Weeks

Deliverables

Web App UI, Dashboard Design

[Hero Web App Mockup: Notty Terminal Dashboard floating on dark background]

The Problem

Decentralized finance (DeFi) platforms and crypto launchpads often suffer from cluttered, highly technical interfaces that alienate new users and increase the risk of costly errors during high-stakes token sales. Information density is extremely high, and bad UI makes it unreadable.

User Insights

Users need extreme clarity on token metrics, vesting schedules, and contribution limits. The primary insight gathered was that trust in Web3 is built directly through transparency and a clean, bug-free visual layout that doesn't overwhelm the user with unnecessary charts.

Identifying the key frustrations of a retail crypto investor who struggles with densely packed interfaces.

User Flow

Project Goals

  • Simplify the dashboard to emphasize key transactional metrics.
  • Design a clear, foolproof step-by-step participation flow for token sales.
  • Establish a modern, high-trust 'terminal' aesthetic.

The Solution

A sleek, dark-mode focused terminal interface that categorizes complex blockchain data into bite-sized, readable widgets. The interface uses minimal but highly intentional color accents to safely guide the user's attention through transactions and portfolio management without visual exhaustion.

Key Design Decisions

📊 Clean Data Visualization

Rather than raw tables of numbers, I utilized modern widgets, progress bars, and high-contrast typography to make financial data instantly scannable.

🛡️ High-Trust UI

Using a strict dark mode palette combined with subtle, crisp borders communicates an institutional level of security, vital for crypto platforms.

⚡ Frictionless Participation

The complex process of connecting a wallet, verifying constraints, and funding a contract was distilled down into a simple linear wizard, preventing costly mistakes.
